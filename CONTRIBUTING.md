# Contributing to Penshort

Thanks for helping improve Penshort. This repository is in Phase 0 (docs and project scaffolding). Application code will be added later.

## Ways to contribute now
- Improve documentation and project templates.
- Propose features or report bugs via the issue templates.

## Proposing changes
1. Check existing issues to avoid duplicates.
2. Open an issue using the appropriate template.
3. For significant changes, wait for maintainer feedback before starting a PR.

## Pull requests
- Use the pull request template.
- Keep changes focused and small.
- Update documentation when behavior or scope changes.
- Add an ADR in `docs/adr/` for significant design decisions.

## Development setup
Development instructions will be added when application code exists. For now, documentation-only changes are welcome.

## Code of Conduct
All contributors must follow [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

## Security issues
Report security issues privately as described in [SECURITY.md](SECURITY.md). Do not open public issues for vulnerabilities.
